JOB PORTAL ‚Äî API INTEGRATION
============================

The Next.js app proxies the job portal API through same-origin BFF routes under
`/api/job-portal/*` and `/api/public/*`.

Configure the upstream API in `.env.local` (local) and in your hosting provider's environment variables (production):

  JOB_API_BASE_URL=https://jobportal.blvcksapphire.com/api/v1

If unset in production, the app defaults to the URL above. Local dev defaults to:

  JOB_API_BASE_URL=http://localhost:8080/api/v1


EMPLOYER PORTAL (/job-portal)
-----------------------------
- Login: email + password ‚Üí 2FA challenge ‚Üí `/auth/2fa/verify` ‚Üí JWT in sessionStorage
- Employer role required (`role = employer`)
- Jobs: GET/POST `/employer/jobs`, GET/PATCH/DELETE `/employer/jobs/:id`
- Applications: GET `/employer/applications`, PATCH `/employer/applications/:id/status`
  (status PATCH includes candidate email fields ‚Äî see below)
- Tokens sent as `Authorization: Bearer <access_token>` via BFF proxy

CANDIDATE EMAIL NOTIFICATIONS
-----------------------------
See `job-portal/CANDIDATE_EMAILS.md` for the full backend ‚Üî frontend contract.

- Status change: frontend composes email and sends with PATCH
  `/employer/applications/:id/status` (`email_subject`, `email_body`; for
  `interview` also `interview_at`, `reminder_email_*`)
- Placeholders expanded client-side; UI: StatusEmailModal on detail + kanban
- Only treat as delivered when response has `email_sent: true`
- `email_sent` not true ‚Üí warning + retry; `503` ‚Üí email not configured
- Mailto links remain available for manual follow-up
- Guest apply success page still notes checking email for a tracking link


PUBLIC SITE
-----------
- Careers: GET `/api/public/jobs` (published jobs from `GET /jobs`)
- Job detail: GET `/api/public/jobs/:id`
- Apply (no account): POST `/api/public/jobs/:id/applications` + POST `/api/public/files/upload-url`
  (upload purposes: `resume`, `document`; apply supports optional `github_url`,
  `additional_link`, `additional_document_url`)
- Per-job required/optional/hidden materials: see `job-portal/APPLICATION_FIELDS_API.md`
  (spec only ‚Äî not implemented yet)
- Legacy candidate portal login remains for existing accounts only (no new registration)
